Autonomous Research Agent
An autonomous, AI-powered research assistant built using LangChain, LangGraph, and the Groq API. This agent is designed to take a given topic, autonomously search the web and Wikipedia for information, and synthesize its findings into a highly structured, professional Markdown report.

🌟 Features
Autonomous Research Workflow: The agent follows a strict, logical sequence to gather and synthesize information without human intervention.

Web & Knowledge Base Integration: Utilizes Wikipedia for foundational background knowledge and DuckDuckGo for the latest news, statistics, and trends.

High-Speed Generation: Powered by the Groq API, enabling incredibly fast inference and report generation.

Structured Output: Automatically compiles unstructured research notes into a polished Markdown report featuring an Executive Summary, Key Findings, Trends, Challenges, and References.

ReAct Agent Architecture: Built with LangGraph's create_react_agent to reason through the task, determine the right tools to use, and formulate the final response.

🛠️ Prerequisites & Dependencies
To run this notebook, you will need a Groq API Key and the following Python packages:

langchain-groq

langchain-community

langchain-core

langgraph

ddgs (DuckDuckGo Search)

wikipedia

🚀 Installation & Setup
1. Install Dependencies
Run the following command in your terminal or Jupyter Notebook cell:

Bash

pip install -q langchain-groq langchain-community langchain-core langgraph ddgs wikipedia
2. Set Up the Groq API Key
You must set your Groq API key as an environment variable. In Python, you can do this via:

Python

import os
os.environ["GROQ_API_KEY"] = "your_groq_api_key_here"
3. Initialize the LLM
The project uses ChatGroq to connect to the Groq inference engine. (Note: The notebook uses openai/gpt-oss-safeguard-20b as a placeholder/safeguard model, but you can configure this to your preferred Llama-3 or Mixtral model available on Groq).

🧠 How It Works
The system utilizes three primary tools:

wikipedia: Queried first to establish baseline background knowledge on the requested topic.

duckduckgo_search: Queried 2-3 times with varied search terms to gather recent developments, market insights, and real-time statistics.

generate_report: A custom tool called last. It takes the combined string of all collected research notes and uses the LLM to format them into a comprehensive technical report.

💻 Usage
You can trigger the research process by passing a string to the research_topic() function:

Python

# Define your topic
topic = "Impact of AI in Healthcare"

# Run the agent
result = research_topic(topic)

# Display the formatted Markdown report
show_report(result)
